ECG-TransNet

Hybrid CNN–Transformer Arrhythmia Intelligence

CNNs see trees. Transformers see forests. Both matter for diagnosis.

Production-grade deep learning framework for automated cardiac arrhythmia classification from single-lead ECG. Fuses morphological precision of 1D-ResNets with long-range temporal reasoning of Transformers. 98.2% accuracy, AAMI EC57 validated, clinical-grade explainability.

Learn More Quickstart →

Trained on MIT-BIH. Validated against AAMI standard. XAI ready.

"Every heartbeat is a sentence. ECG-TransNet reads the paragraph."

Traditional approaches treat arrhythmia detection as local pattern recognition (pure CNN) or global sequence understanding (pure Transformer). But cardiac diagnosis requires both: the QRS morphology matters, and the R-R interval rhythm matters. ECG-TransNet unifies these paradigms.

Three-Stage Architecture

ECG-TransNet is a unified end-to-end pipeline that captures both morphology and rhythm:

Raw ECG Signal (360 Hz)
        │
        ▼
┌─────────────────────────────────────────┐
│   STAGE 1 · CNN Feature Encoder         │
│  [ResBlock 64] → [ResBlock 128]        │
│              → [ResBlock 256]          │
│  Shift-invariant 1D convolutions       │
│  Captures: QRS, ST-segment, T-wave     │
└─────────────────────────────────────────┘
        │ Token sequence [t₁, t₂, ..., tₙ]
        ▼
┌─────────────────────────────────────────┐
│   STAGE 2 · Transformer Encoder         │
│  Multi-Head Self-Attention (8 heads)    │
│  Q · Kᵀ / √d → softmax → V             │
│  Learns global R-R interval patterns    │
│  4 encoder layers + FFN + LayerNorm     │
└─────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│   STAGE 3 · Classification Head + XAI   │
│  Global Avg Pool → Linear(256→5)        │
│  Grad-CAM saliency overlay              │
│  SHAP feature attribution               │
└─────────────────────────────────────────┘
        │
        ▼
  AAMI Class: {N, S, V, F, Q}

Why This Hybrid Approach?

Approach Strength Weakness ECG-TransNet
Pure CNN Fast, interpretable, local patterns No long-range context Stage 1: CNN encoder
Pure Transformer Global attention, rhythm awareness Expensive, loses morphology Stage 2: Transformer encoder
Hybrid (ECG-TransNet) Both local + global in one pass Balanced complexity ✓ 98.2% accuracy, <12ms

Explainability (XAI)

Every prediction ships with clinical-grade explanations:

🎯 Grad-CAM Saliency

Identifies which ECG signal regions (QRS, T-wave, etc.) triggered each classification decision. Heatmaps overlay directly on waveforms.

📊 SHAP Attribution

Per-sample feature attribution across 360 dimensions. Shows which micro-features (peak amplitude, slope, intervals) drove each prediction.

Clinical Trust: FDA AI/ML guidance and EU MDR increasingly require human-interpretable justifications. ECG-TransNet's XAI suite was designed from day one to satisfy regulatory bar.

Quickstart

1. Install

git clone https://github.com/prakulhiremath/ECG-TransNet-Arrhythmia-Detection
cd ECG-TransNet-Arrhythmia-Detection
pip install -r requirements.txt

2. Download Data

python data/download_mitbih.py --output data/raw/

3. Preprocess

python preprocessing/pan_tompkins.py --input data/raw/ --output data/beats/

4. Train

python training/train.py --config configs/production_run.yaml

5. Explain Predictions

# Grad-CAM heatmap
python interpretability/gradcam.py --sample_id 100_beat_5

# SHAP attribution
python interpretability/shap_analysis.py

Dataset: MIT-BIH Arrhythmia Database

Source: PhysioNet
Recordings: 48 half-hour, dual-channel, 360 Hz
Beats: ~110,000 manually annotated
Standard: AAMI EC57 (5-class remapping)

Class Distribution Challenge

Class Description Count Share
N Normal + Bundle Branch ~90,000 ~82%
S Supraventricular Ectopic ~2,700 ~2.5%
V Ventricular Ectopic ~7,200 ~6.6%
F Fusion Beats ~800 ~0.7%
Q Unknown/Paced ~8,000 ~7.3%

The Long-Tail Problem: F-class has 112× fewer samples than N-class. ECG-TransNet addresses this with class-weighted loss and beat-morphology augmentation.
